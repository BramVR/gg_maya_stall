# Mirror Crabbox Visual Evidence Policy

Maya Stall runs must produce Visual Evidence using the same policy shape as Crabbox where the Session Broker can prove it: screenshots are expected, and recordings use Crabbox-like desktop frame capture plus local MP4 encoding. A full Evidence Bundle can include screenshots, recordings, logs, metadata, scenes, and output files. Real Maya UI failures are too hard to diagnose from logs alone, so Visual Evidence is part of the core evidence model rather than optional polish.
